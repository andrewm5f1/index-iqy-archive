# index-iqy-archive

本仓库用于归档和发布多个独立 HTML 页面。这些页面不针对任何单个域名或具体网站，而是作为通用存档用途。

## 项目简介

index-iqy-archive 是一个静态 HTML 页面归档仓库。仓库中的页面内容以 HTML 文件形式存储，可通过 GitHub Pages 或其他静态托管服务直接访问。

## 目录说明

```
/
├── index.html         # 仓库首页 / 导航页面
├── pages/             # 归档页面存放目录
│   ├── example.html
│   └── ...
└── assets/            # 公共资源（CSS、JS、图片等）
    ├── style.css
    └── ...
```

- `index.html`：仓库主页面，通常包含导航或说明信息。
- `pages/`：存放主要归档页面文件。每个页面为独立 HTML 文件。
- `assets/`：存放页面所需的公共静态资源，如样式表、脚本、图片等。

## 页面归档说明

- 每个归档页面均为独立、完整的 HTML 文件，可以直接在浏览器中打开。
- 页面内容不包含任何指向具体网站或域名的链接，也不包含营销性质的内容。
- 归档内容仅用于资料存储与查阅。

## 维护说明

- 本仓库由维护者不定期更新。
- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并更新 `index.html` 中的导航（如适用）。
- 所有提交应保持页面完整性，避免依赖外部资源（除非已包含在 `assets/` 中）。
- 欢迎提交 Issue 或 Pull Request，但需确保内容符合仓库定位。
